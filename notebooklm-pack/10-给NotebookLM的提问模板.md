# 给 NotebookLM 的提问模板

这份文件的目的，是让你上传后可以直接复制问题去问 NotebookLM。

## 1. 让它帮你拆学习计划


```text
请根据这些源码讲解文档，为我定制一份“3天高强度突击”速成计划。
要求：
1. 压缩篇幅：舍弃边缘功能和非核心逻辑，每天内容直击底层架构和数据流。
2. 精准定位：每天明确列出 3-5 个必须读透的核心源文件，并说明它们之间的调用关系。
3. 实战练习：每天的练习改为“代码重构”或“功能复刻”，侧重于手动实现核心链路。
4. 避坑自测：自测题需聚焦于真实开发中高频出现的报错场景（如数据库连接失败、异步处理错位等）。
```

## 2. 让它讲项目整体结构

```text
请根据这些资料，用初学者能听懂的方式解释这个项目的整体结构。
重点讲清楚：
1. server.js 做什么
2. routes 做什么
3. controllers 做什么
4. models 做什么
5. 前端如何调用后端
```

## 3. 让它只讲一个文件

```text
请只讲 server.js，不要讲其他文件。
要求：
1. 逐行解释
2. 解释每一段的作用
3. 用通俗中文讲
4. 给我 5 个自测问题
```

## 4. 让它帮你理解请求流转

```text
请结合这些资料，详细解释一个“创建食物”的请求是如何流转的。
要求按这个顺序讲：
Request -> Route -> Controller -> Model -> DB -> Response
```

## 5. 让它给你出练习题

```text
请根据这个项目给我出 10 道练习题。
要求：
1. 题目要循序渐进
2. 不要直接给答案
3. 每道题告诉我应该看哪个文件
4. 题目要覆盖增删改查和前后端对接
```

## 6. 让它检查你是否真的理解了

```text
我现在学完了 routes 和 controller。
请你像面试官一样问我 10 个问题，检查我是否真的理解。
要求：
1. 不要太难
2. 一次只问一个
3. 等我回答后再继续下一个
```

## 7. 让它带你做功能扩展

```text
请基于这些项目资料，指导我新增一个 GET /food/:id 接口。
要求：
1. 不要直接给完整答案
2. 先告诉我应该改哪些文件
3. 再一步一步带我实现
4. 每一步都解释为什么要这样改
```

## 8. 让它对比概念

```text
请结合这个项目，帮我理解：
1. req.body
2. req.params
3. req.query

要求：
1. 结合当前项目里的真实接口举例
2. 不要只给抽象定义
3. 要告诉我分别在什么场景使用
```

## 9. 让它把知识整理成笔记

```text
请把这个项目整理成一份适合初学者复习的笔记。
要求：
1. 按模块分节
2. 每节只讲最关键的点
3. 用简洁中文
4. 最后给一份总复习清单
```

## 10. 让它帮助你避免依赖答案

```text
接下来你不要直接给我完整代码答案。
你需要像老师一样带我做：
1. 先提示思路
2. 再告诉我应该看哪个文件
3. 再让我自己尝试
4. 如果我卡住了，你再给我下一步提示
```

## 最后建议

你问 NotebookLM 时，尽量带上这三类要求：

- 讲给初学者听
- 不要一下讲太多
- 结合当前项目真实代码解释

这样它给你的内容会更适合学习，而不是只给抽象概念。
