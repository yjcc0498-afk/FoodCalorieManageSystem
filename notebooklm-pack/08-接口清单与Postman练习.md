# 接口清单与 Postman 练习

## 1. 创建食物

### 请求

- Method: `POST`
- URL: `http://localhost:3000/food`

### Body

```json
{
  "name": "Chicken Breast",
  "calories": 165
}
```

### 你应该观察什么

- 返回状态码是否是 `201`
- 返回的 `data.name` 是否被转成小写
- 返回的 `createdAt` 和 `updatedAt` 是否自动生成

## 2. 获取全部食物

### 请求

- Method: `GET`
- URL: `http://localhost:3000/foods`

### 你应该观察什么

- 返回状态码是否是 `200`
- 是否有 `count`
- `data` 是否是数组

## 3. 按名称搜索食物

### 请求

- Method: `GET`
- URL: `http://localhost:3000/foods?keyword=chicken`

### 你应该观察什么

- 是否只返回匹配的食物
- 返回中的 `keyword` 是否等于你搜索的值

## 4. 更新食物热量

### 请求

- Method: `PATCH`
- URL: `http://localhost:3000/food/:id`

### Body

```json
{
  "calories": 180
}
```

### 你应该观察什么

- 返回状态码是否是 `200`
- 返回的 `data.calories` 是否更新成功

## 5. 删除食物

### 请求

- Method: `DELETE`
- URL: `http://localhost:3000/food/:id`

### 你应该观察什么

- 返回状态码是否是 `200`
- 返回的 `data` 是否是被删除的那条记录

## 6. 错误练习建议

为了真正学会接口，你不要只做成功案例，也要做失败案例。

### 练习 1：创建时不传 `name`

观察：

- 状态码是不是 `400`
- 返回信息是否清晰

### 练习 2：创建时传负数热量

观察：

- 状态码是不是 `400`
- 为什么 controller 会拦截

### 练习 3：更新时传非法 ID

观察：

- 状态码是不是 `400`
- 为什么不是 `404`

### 练习 4：删除一个不存在的合法 ID

观察：

- 状态码是不是 `404`
- 和非法 ID 的区别是什么

## 7. 你要掌握的不是接口地址，而是接口思维

每测一个接口都问自己：

1. 请求参数从哪里传
2. 后端在哪里接收
3. 在哪里校验
4. 在哪里操作数据库
5. 响应数据结构长什么样
