# 项目总览与学习路线

## 1. 这个项目是做什么的

这是一个食物热量管理系统的后端 MVP，并补了一个简单前端页面。

它的核心功能是：

- 新增食物
- 查询所有食物
- 按名称搜索食物
- 更新食物热量
- 删除食物

你可以把它理解成一个很典型的“前后端分层练习项目”。

## 2. 为什么这个项目适合初学者

这个项目小而完整，正好覆盖了后端开发里最重要的一条主线：

1. 浏览器或 Postman 发请求
2. Express 路由接收请求
3. Controller 做参数校验和业务处理
4. Model 负责字段规则
5. MongoDB 保存数据
6. 结果再返回给前端

这就是很多 Web 项目的基础骨架。

## 4. 每层是干什么的

如果你有 JS 逆向或爬虫思维，可以这样理解：

- `routes`
  像请求包入口分发器，决定这个请求该走哪条链路
- `controllers`
  像参数组装、发包、响应解析层，负责业务处理
- `models`
  像字段协议定义层，规定数据库里数据长什么样
- `config/db.js`
  像底层连接初始化模块，只负责连接数据库
- `server.js`
  像总装配入口，把所有模块拼起来
- `public/app.js`
  像前端请求调度器，负责调用后端接口并更新页面

## 5. 当前项目实现了哪些接口

- `POST /food`
- `GET /foods`
- `GET /foods?keyword=chicken`
- `PATCH /food/:id`
- `DELETE /food/:id`

## 6. 建议学习顺序

对于初学者，不建议一上来就看 `controller` 全部细节。

推荐顺序：

1. 先看 `server.js`
2. 再看 `routes/foodRoutes.js`
3. 再看 `controllers/foodController.js`
4. 再看 `models/Food.js`
5. 再看 `config/db.js`
6. 最后看 `public/app.js`

原因：

- 先从“请求从哪里进来”开始，比较容易建立全局视角
- 然后再看“请求在内部怎么流动”
- 最后再理解前端如何把接口用起来

## 7. 适合你的阶段性目标

### 第一阶段：会运行

你要能做到：

- 启动 MongoDB
- 启动 Node 服务
- 打开页面
- 能完成增删改查

### 第二阶段：会解释

你要能说清楚：

- 为什么要分 `routes / controllers / models`
- 为什么数据库连接放在 `config`
- 一个请求是怎么从浏览器走到 MongoDB 的

### 第三阶段：会修改

你要能自己改出这些功能：

- 新增单条查询接口
- 新增按热量范围筛选
- 新增更新食物名称功能
- 新增错误提示优化

## 8. 学习时要重点盯住的问题

看代码时不断问自己：

- 请求从哪里进来？
- 参数在哪里取？
- 参数在哪里校验？
- 数据在哪里写进数据库？
- 响应在哪里返回？
- 为什么这个逻辑放在这里而不是别处？

只要你能持续围绕这 6 个问题看代码，进步会非常快。
