# 初学者任务拆解

下面是一套非常适合初学者的任务拆法。

你可以自己照着做，也可以把这份文档上传后，让 NotebookLM 按这个风格继续拆。

## 第 1 组任务：先跑通项目

目标：

- 先把项目运行起来

任务：

1. 启动本地 MongoDB
2. 检查 `.env`
3. 运行 `npm start`
4. 打开 `http://localhost:3000/`
5. 在页面里新增一条食物记录

你应该学到：

- 项目是如何启动的
- 数据库连接和服务启动的关系

## 第 2 组任务：搞懂请求流转

目标：

- 看懂一个请求从哪里来、到哪里去

任务：

1. 看 `server.js`
2. 看 `routes/foodRoutes.js`
3. 看 `controllers/foodController.js`
4. 看 `models/Food.js`
5. 画出 `Request -> Route -> Controller -> Model -> DB -> Response`

你应该学到：

- 项目分层思维
- 后端请求处理流程

## 第 3 组任务：重点学创建接口

目标：

- 看懂新增一条食物的全过程

任务：

1. 用页面新增一条记录
2. 用 Postman 再新增一条记录
3. 只追踪 `POST /food`
4. 说明 `name` 和 `calories` 在哪里被校验

你应该学到：

- `req.body`
- 参数校验
- `Food.create`

## 第 4 组任务：重点学查询接口

目标：

- 搞懂查询全部和模糊搜索

任务：

1. 测试 `GET /foods`
2. 测试 `GET /foods?keyword=chicken`
3. 找到 `keyword` 在哪里被读取
4. 找到 MongoDB 查询条件是如何拼装的

你应该学到：

- `req.query`
- 查询条件构造
- 模糊搜索

## 第 5 组任务：重点学更新和删除

目标：

- 看懂 `params`、`ObjectId` 校验、数据库更新

任务：

1. 用页面更新热量
2. 用页面删除一条记录
3. 找出 `req.params.id` 在哪里被读取
4. 找出为什么先校验 `ObjectId`

你应该学到：

- `req.params`
- `findByIdAndUpdate`
- `findByIdAndDelete`
- `400` 和 `404` 的区别

## 第 6 组任务：开始自己改功能

目标：

- 从“看懂”进入“会修改”

建议功能：

1. 新增 `GET /food/:id`
2. 新增更新食物名称功能
3. 新增热量范围筛选
4. 在前端页面显示 `updatedAt`

## 第 7 组任务：整理你的知识卡片

目标：

- 不只是做完，而是沉淀方法

请自己写出：

1. `server.js` 的职责
2. `route` 和 `controller` 的区别
3. `model` 的作用
4. `req.body / req.params / req.query` 的区别
5. 一个接口完整流转过程

## 最重要的提醒

不要同时学太多文件。

推荐策略：

- 一次只盯一个接口
- 一次只追一条调用链
- 一次只搞懂一个核心概念

这比“从头到尾把所有代码都扫一遍”有效得多。
